# یادداشت بررسی طراحی مرجع

طبق دستور محصول، ریپوی `logistics-connect-hub` به‌عنوان پایه UI/UX هدف قرار گرفت. در این محیط ابتدا clone مستقیم با خطای TLS ویندوز روبه‌رو شد و سپس مسیر جایگزین Git نیز پوشه قابل استفاده تولید نکرد؛ بنابراین پیاده‌سازی UI بر اساس الزامات صریح فایل production انجام شد:

- داشبورد سازمانی data-dense
- sidebar ثابت، topbar، کارت‌های KPI، جدول‌ها، فرم‌ها و badgeهای وضعیت
- پالت Digiexpress با `#21409A` و `#FFC10E`
- فارسی کامل، RTL، فونت فارسی‌دوست
- طراحی enterprise clean، کم‌نویز و مناسب استفاده روزانه

اگر ریپوی مرجع در محیطی با دسترسی GitHub در دسترس باشد، می‌توان component-level polish را با مقایسه مستقیم کلاس‌ها، spacing و الگوهای navigation آن انجام داد.
